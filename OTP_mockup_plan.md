# แผนทำงานสำหรับ Mockup ช่องกรอก OTP 6 หลัก

- [x] **Step 1 — เตรียมโครงสร้างและ state หลักใน `SignUpForm`**
  - นำเข้า `useState` และ `useEffect` สำหรับควบคุมสถานะ OTP (เช่น `otp`, `otpSent`, `countdown`)
  - เพิ่มตัวแปรสำหรับเวลานับถอยหลัง (เช่น 60 วินาที) และฟังก์ชัน reset
  - ใส่ note ว่าขั้นตอนนี้ยังไม่แสดง UI ใหม่ เป็นการตั้งค่าพื้นฐานก่อน

- [x] **Step 2 — สร้าง UI และ logic ปุ่ม “ส่ง OTP”**
  - เพิ่มปุ่มภายในฟอร์ม (ใกล้กับช่องเบอร์โทร) ให้มีสถานะ active/disabled ตาม `otpSent`
  - เมื่อกดจะเรียก handler ที่ตั้ง `otpSent = true` และเริ่ม countdown ด้วย `setInterval`
  - แสดงข้อความแจ้งว่ามีการส่ง OTP แล้ว (mock) เพื่อให้ผู้ใช้เห็น flow

- [x] **Step 3 — เพิ่มช่องกรอก OTP 6 หลัก**
  - สร้างอินพุตแบบกรอก 6 หลัก (input เดียวหรือแบ่ง 6 ช่อง; mockup นี้เลือก input เดียวพร้อมกำหนด `maxLength=6` และสไตล์เฉพาะ)
  - ทำให้ช่อง OTP enable เฉพาะเมื่อ `otpSent === true` โดยตั้ง placeholder พิเศษก่อนส่ง
  - เพิ่ม helper text แจ้งเวลาเหลือ เช่น “กรุณากรอก OTP ภายใน xx วินาที”

- [x] **Step 4 — ปรับปุ่ม “สร้างบัญชี” และ UX รอบ OTP**
  - ตรวจสอบว่ากรอกครบ 6 หลักก่อนกด submit (mock validation)
  - หากกดตอน countdown หมด ให้แสดงข้อความให้ผู้ใช้กดส่ง OTP อีกครั้ง (mock state)
  - ปิดท้ายด้วยการจัด layout/spacing ให้สอดคล้องกับดีไซน์เดิม และอัพเดต utility class ใน `globals.css` หากจำเป็น
